Pinky: A Compound AI System on Consumer Hardware

A five-stage pipeline (cache → classifier → dispatcher → specialists → verifier) running across nine microservice agents on three compute tiers. Built because no single model is reliably metacognitive about its own ability to answer.

TL;DR

Pinky started life on April 15 as a persona chatbot in Open WebUI talking to a single 20B model via Ollama. Five weeks later it is a compound AI system in active rebuild: dispatcher, specialist ensemble, decorrelated verifier, hybrid RAG with a grounding gate. The thing that drove every architectural shift was the same lesson, learned harder each time: no single model is reliably metacognitive about its own ability to answer.

How it started

The earliest commits stood up a single 20B model behind a persona prompt, with SearXNG for web search and a vector lookup against an Obsidian vault. The "multi-agent" part was a regex-based router in a pipelines shim that pattern-matched on the user message and string-templated a delegation to another persona-flavored prompt running on the same model.

This was one model, one prompt, light tool use. The "agents" were system-prompt costumes the same model wore. The router was a regex, not a classifier.

It worked well enough to demo. It did not work well enough to be honest.

Phase 1: Multi-persona chatbot to real microservices

The persona-swap inside one model bled. Pinky would slip into another persona's voice mid-response. The pipelines shim could not run real async ReAct loops; it was a streaming text proxy with extra steps.

The move was to tear out the shim and give every agent its own FastAPI plus LangGraph service with its own ReAct loop, tool surface, and model assignment. Pinky became a real orchestrator with a planner LLM call instead of a regex.

This phase ended with fourteen microservice agents running, a FalkorDB knowledge graph, a Qdrant vector store, and an Obsidian-as-memory pattern where the Archivist agent owned all writes.

It was more correct than the regex shim. It was also more agents than the planner could keep straight.

Phase 2: Microservices to a designed compound system

This is where the architecture stopped being "more agents" and started being a system designed around an orchestration pattern. Four shifts in two weeks.

Fourteen agents consolidated to nine. Duplicate fan-outs collapsed into mode-switched specialists. QA and Scrum Manager merged into Critic. Code Writer and Release Manager merged into Developer. Tron and Rinzler merged into Security. Video Editor, Audio, and Highlighter merged into Studio. The Surrealist was developed to add useful entropy — the engine behind the Mucha-swerve technique. Too many agents had been creating routing ambiguity. Decomposition is not a virtue when the planner cannot tell two near-duplicates apart.

Mucha-line async redesign. The previous concurrency model used an is_free pre-dispatch check that livelocked under multi-agent fan-out. Replaced with unconditional dispatch, per-agent outbox, and a 180-second timeout-driven absorb. The concurrency model was not actually real until this point.

Meta-router. A tiny granite4:3b-h JSON classifier on the iGPU emits a six-bucket intent classification in under 300 milliseconds: vault-touching, closed-reason, capability gap, clarify, trivial, multi-agent. The planner branches off the bucket instead of always running a full plan. The planner LLM had been getting called for "tell me a joke." Wasteful, and a hallucination surface. Lightweight classifier in front fixes both.

RAG, properly. What had existed before was retrieval tools that an agent might or might not call. RAG is a loop, not a tool. The new implementation: section-aware chunking, Qdrant v2 collection with hybrid (BM25 plus vector) retrieval fused via Reciprocal Rank Fusion, bge-reranker-v2-m3 cross-encoder reranking, a dedicated _handle_vault_touching node, and a grounding validator that checks the answering model against a different model family. Ungrounded answers had been a real failure mode; structurally eliminated.

In parallel, Ollama got replaced with llama-server plus LiteLLM as the inference layer. Single runner family across CUDA, ROCm, and CPU. LiteLLM gateway exposes role aliases (pinky-main, osint-main, critic-main, and so on). Symlinks into the existing blob store, no re-quantization required.

By the end of this phase the shape of a compound AI system was visible. The orchestrator was still a "chief-of-staff" pattern, though. Pinky's LLM decided when to self-handle versus dispatch, and a closed_reason path let it short-circuit lookups when it thought it already knew the answer.

That last part is where the next failure lived.

Phase 3: The chief-of-staff wound

The triggering failure was a GAIA L1 question: "What writer is quoted by Merriam-Webster on 2022-06-27?"

Full-pipeline runs got it right. Annie Levin. The closed_reason shortcut, where Pinky's planner decided "I know this, no lookup needed," fabricated Oscar Wilde confidently.

Same model. Different routing decision. Either correct or hallucinated.

That failure sits on top of four weeks of failures with the same shape:

  • closed_reason hallucinations on lookup-shaped questions.
  • Chief-of-staff over-delegation: three parallel Surrealist calls for one prompt.
  • Chief-of-staff under-delegation: dropping the Developer dispatch on "make a hello world C app."
  • Four rebenches across cognitive roster, meta-router, and two general re-benches. All trying to find a "magic chief" model that would route correctly. None converged.

The diagnosis, codified in ADR-002: the chief role itself is the wound. No single LLM is reliably right about its own ability to answer. Swapping the chief swaps the failure mode; it does not remove it.

Priority order codified in ADR-002: Stability > Responsiveness > Parallelism > Accuracy. A fast wrong answer is worse than a slow correct one.

Hardware

Tier Hardware Role
iGPU Radeon 780M, 4 GB UMA Stage-1 classifier only. Pinning is role assignment, not a residency trick.
eGPU RTX 5060 Ti, 16 GB Planner, critic, creative specialist, verifier. Migrating to TRT-LLM co-residency for true concurrent residency.
CPU (host) Embeddings (nomic-embed, BGE-large), reranker (bge-reranker-v2-m3), router classifier, studio (qwen3:8b), fallback inference.

That is the whole rig. No cloud burst yet; designed in as an escalation path, not a primary.

Why compound AI, distilled

Metacognition is the wound. A single model deciding whether it knows the answer is the failure surface every chief-of-staff pattern hits. Staging the decision across classifier ≠ answerer ≠ verifier removes the self-doubt loop.

Decorrelation buys real entropy. ADR-002 makes "verifier must be a different model family from the answerer" a hard rule. Same family means same blind spots means the verifier rubber-stamps. Mistral or Gemma or Phi verify Qwen; the families are chosen for decorrelated error distributions, not for individual benchmarks.

Grounding gate kills hallucination cliffs. Every closed_reason answer must trace a citation. No source means escalate, not ship. Eliminates the Oscar Wilde class of failure structurally.

N-furcation beats model swapping. When no single model is reliably right on ambiguous or high-stakes questions, racing two or three specialists and taking consensus is cheaper than another bench cycle. Costs about 1.5 to 2 times the inference on roughly 20 percent of queries; consistently right.

Compute economics on consumer hardware. The same model file serves multiple roles via system prompt, tool set, and per-request flags. Planner is reasoner with enable_thinking=true. Four or five model files in flight, not one per role. Critical on a 16 GB GPU.

Modal expansion fits the pattern. Vision and audio specialists slot in as new worker classes. Stage 1 classifies modality, dispatcher routes. The architecture survives growth.

Rejected: "just find a better chief." Four weeks of benches proved that swapping the chief swaps the failure mode rather than removing it. ADR-002 calls this out explicitly so future me does not waste a fifth week.

What's shipped, what's in flight

Shipped: All five stages of the staged-router pipeline as a runnable code path. Verifier is wired into the _handle_closed_reason path (the Oscar Wilde fix). Speculative decoding live on the CUDA router with measured results: 22 percent throughput lift, 60 percent draft acceptance. Twenty-nine integration tests green. Hybrid RAG and grounding validator live.

In flight: Making StagedRouterPipeline the default routing path across all intents, not just closed_reason. Observation-mode A/B between staged and legacy. Specialist wrappers for the researcher, code, security, and agent services. TRT-LLM co-residency cutover for the eGPU tier.

Not yet: Vision specialists. Audio specialists. Voice loop (VAD-CPU → ASR-eGPU → LLM-eGPU → TTS-CPU). Cloud-burst escalation.

What this should tell you about how I work

Pinky began as a persona over a single 20B model and ended up, by repeatedly losing the same fight, as a dispatcher into a specialist ensemble guarded by a decorrelated verifier. The compound AI shape is not a design preference. It is the structural answer to "no single LLM is reliably honest about what it does not know."

The interesting part is not the architecture diagram. It is that the architecture diagram is the fourth one. Each version solved the previous version's stated problem and revealed the next problem underneath it. That is what good architecture work actually feels like.
